Synteny

Run MCScanX for synteny analysis. It is not super easy to use and a bunch of stuff is hardcoded that is not mentioned anywhere in the documentation.

Toggle down for details on all the problems I had MCScanX is based on protein synteny from blastp searches. The documentation is not super useful beyond the installation instructions, so I was recommended this workflow, which I will use for detailed instructions, but for preparing the input files, I am using my own scripts and not their provided ones (esp. the perl scripts have hard-coded aspects that assume ncbi assemblies and annotations, which doesn't apply to my data). MCScanX only takes one input: the prefix of both files below. It says directory in the documentation but that's wrong. You run from the same directory as the files are in and the command is just ./MCScanX prefix where the input files are prefix.gff and prefix.blast in the same directory.
  • prefix.gff: I used my script src/make_bedfile_for_MCScanX.py which makes all the files according to the format required, including reformatting the contig names, and also creating a lookup table to associate the new contig names with the original ones from the annotations. The files are then all merged with cat *.bed > prefix.gff
  • prefix.blast: I am using blastp according to the above cited workflow on all the species bash/blastp_for_synteny.sh. All blast results are again merged cat *.blast > prefix.blast.
If the error message is no alighments generated: The log file reads no blast alignments. For me, this was caused by the file extension of the annotation file. I named it .bed but it should be .gff (because it is somehow hardcoded and there is no test to see if a file actually exists?)

dN/dS analysis

paml codeml

I will run codeml as is specified in the manual for pairwise analyses with runmode= -2, CodonFreq = 2. In order to properly parallelize this the scripts are nested like this:

  1. src/blast_BRH/calculate_pairwise_dNdS.py is the main script that runs the pipeline based on one input fasta file
  2. bash/run_batch_dNdS_A.sh (or X) loops over a list of input files to run the main script
  3. src/blast_BRH/calculate_batch_pw_dNdS.py separates the entire list of input files into several batches to run in parallel
  4. bash/run_python_wrapper_for_batch_dNdS.sh runs the batch-separation script as a slurm job since it can be quite computationally intensive for long nested files lists

This launches a long list jobs, one for each batch of orthologs, that all run in parallel. Each job individually should all take less than an hour. I have separated everything for A and X into its own directories, so the outermost bash script bash/run_python_wrapper_for_batch_dNdS.sh has to be run from either the A or X species directory and then also pass the corresponding "A" or "X" command line argument to the batching python script. This results in a separate directory for each ortholog where the dN and dS values are in separate files called 2ML.dN and 2ML.dS, and the script calculates a dNdS value already into a separate file called 2ML.dNdS. Since this does not take any filtering for maximum dN and dS value into account that we apply later, this will not be used in the future.

the dN and dS values can be summarized into one list using src/blast_BRH/extract_dNdS_Results.py (and bash/run_extract_dNdS_results.sh to run it as a slurm job since this also takes a bit to loop through all the autosomes). This again results in two separate files, one for A and one for X orthologs.

Data preparation and visualization

We hypothesize that the evolutionary rates are different for older vs. younger genes and therefore incorporate a measure of gene age (from the perspective of C. maculatus into our analyses). It describes what species is the most distant to still contain a 1-to-1 ortholog: a gene of age 1 is the youngest and the ortholog is only shared with the closest sister species of C. chinensis, while a gene age of 5 is the most conserved with an ortholog in D. melanogaster.

statistical analysis

Median quantile regression with wald test to drop explanatory variables and see if they make a significant difference. This way We can get a p-value for just the gene age (and gene age by chromosome interaction) without worrying about the intercept for interpretation. Further, the results below do agree with the permutation test and show a significantly lower dNdS on X, but it also takes the gene age effect into account.

The code is also in src/plotting/analyze_dNdS_LFC.py, in the function statistical_analysis_dNdS(... , include_sex_bias=False). Since the response variable is dNdS, the analysis can be done from the perspective of C. maculatus with each of the three other Bruchini species.

statistical analysis

The statistical analysis here is a logistic regression with positive selection as the categorically binary response variable to infer an effect of gene age and chromosome.
