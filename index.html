paml branch models: dN/dS

Pairwise estimation with codeml is fine according to the manual above, I should not use YN00, the question is only about the model settings in the config file.

  • pairwise according to manual: runmode= -2, CodonFreq = 2 (for information on runmode = -2 see p. 35 in the paml manual linked above (my script already implements the shortening of species names mentioned there).
  • pairwie how i did it: runmode= 1, CodonFreq = 2 which reads a tree from a tree structure file I generate with FastTree from the pairwise alignment. I think I do it this way because the script was originally intended for multi-species comparisons and when we decided to restrict to pairwise I never changed it. (for information on runmode = -2 see p. 19 in the paml manual)

I think that these will give very similar results, because I hypothesize that runmode= 1 with a two-species tree collapses into runmode= -2 with maybe some rounding error. I implemented the --branch_pairwise flag (untested!) in the pipeline script to run this. I will run on a subset of the orthologs for the normal branch model and test a correlation to see how similar they are. If it's not very similar, I can re-run the entire thing in this mode and re-do the analysis. It would be a bit annoying to update all the SI tables but the rest is pretty automated.

paml site models: positive selection

The reviewer is a bit vague here, but we interpret their comments to mean that they think the pairwise comparison does not offer enough statistical power to accurately infer positive selection, and that we would need multi-species comparisons for that. the site models are described on p. 29 of the above manual, and I can find no mention that there is a minimum number of species required for sufficient statistical power. It may depend on the amount of sequence divergence as well. I will test the validity of this argument similar as above, by re-running a smaller test set of orthologs. In order to determine the accuracy of our pairwise method, I need to identify which orthologs are both pairwie 1-to-1 and phylogeny-wide 1-to-1. If my method is correct, I expect to find phylogeny-wide positive selection if at least one pair shows positive selection.

A good description of paml useage is in the methods of Krasovec et al. (2018), but since they do multi-species comparison they use a clustering approach to identify the 1-to-1 orthologs.

  1. identify phylogeny-wide 1-to-1 orthologs and associate with ortholog pairs
  2. run the site model for the phylogeny-wide pairs
  3. associate the new positive selection results with the original pairwise results
  4. optional: correlate the mean ω0 values of all the original pairs with the single ω0 values of the phylogeny-wide orthologs

This is not as certain. I am unsure if it is even accurate to compare the orthologs in this way, and if it's really a dealbraker if the results are inaccurate. If i use the site-model for only the most conserved orthologs, I would have to re-do the ortholog inference probably with the clustering approach from Krasovec et al. instead of the blast BRH method.

I might also try the branch-site model, where i specify a foreground branch to compare to the new pairwise ortholog. However you are really not supposed to just test every possible foreground branch for detection of positive selection like i would have to, but instead have a prior hypothesis of one or a few to try that are biologically interesting.
